# 口味档案 TASTE(甲方画像)

> 给 AI 的:用户没明说的决策(配色/节奏/密度/边界感),以这里为准。
> 每条带【来源】;标【推断】的是从行为猜的,可随时删。冲突时:用户当次明确要求 > 本档案 > 通用套餐。

## 视觉基调

- 深色优先:底色近黑(#0c~#10 段),文字米白,一页一个强调色【来源:全库 demo 与样板间沿用无异议】
- 强调色走荧光系(荧光绿 #c5f542、紫 #8b7cf6 这一档),不用低饱和莫兰迪【推断:多 demo 沿用未被反对】
- 章节标题用 `//` 注释风 + 等宽字体,程序员审美【来源:comment-heading 是用户点名"最快落地"的效果;样板间沿用】
- 文案中文、短句、有态度("效果不是装饰,是语言")【来源:仓库全部文档风格】

## 动效节奏

- 默认 token:`--dur-s .3s / --dur-m .6s / --dur-l 1s`,缓动 `cubic-bezier(.22,1,.36,1)`【来源:样板间实装,用户验收】
- **利落 > 互溶**:过渡的中间帧必须"干净"——要么 A 要么 B,边界扫过,不许两态糊成叠影【来源:image-displacement 泥色叠影被点名不满意,2026-07-16】
- 宁少勿多:2-4 个效果 + 1 个记忆点;强注意力效果(波纹/marquee/displacement/粒子)一页一次【来源:INDEX 铁律,用户认可口径】
- 入场要有编排感:组合页里 Enter/首屏是全页动效的 t=0,不许各效果抢跑【来源:样板间实战笔记】

## 工程口味

- 零依赖单文件是默认;能 CSS 不 JS,能一行不十行【来源:仓库硬规范 + 用户长期开 ponytail 模式】
- 旋钮必须收口:时长/缓动走 token,强度参数进文件顶部 KNOB 常量块带范围注释;改"整体快/慢"只动一处【来源:铁律 7,用户发起并拍板】
- 简化要标注天花板和升级路径(如:无状态粒子 → FBO)【来源:particle-transition 简化版被接受的方式】
- 验证优先于宣称:交付前语法检查 + 行为探针,说得出证据【推断:用户接受了历次"定量验证"的工作方式】

## 黑名单(别做)

- 全屏线性 crossfade 的泥色中间态【来源:2026-07-16 判词】
- 项目已有动效 token 体系时另起一套【来源:铁律 7 对抗审查】
- 声音自动播放(必须手势解锁)、hover 无触屏/键盘等价、reduced-motion 不降级【来源:铁律 2/3/4】
- 为展示"技术力"堆效果——8 个效果打架不如 3 个效果同频【来源:INDEX 选型原则】

## 维护规则

- 用户说"不满意/不喜欢" → **判词落这里**(记"为什么不行");技术坑落对应 pattern README 的"实战笔记"——两个去向别混
- 新判词和旧条目冲突时,新的覆盖旧的,旧条目删掉不留尸体
